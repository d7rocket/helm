# HELM

## Design Context

### Users
A solo operator at a dark desk, commanding a fleet of Claude Code agents across many projects. Jobs: review what the fleet did today, fire automations without typing litanies, audit any past session.

### Brand Personality
Commanding, mechanical, calm. A declassified flight-ops room, not a SaaS dashboard. Never neon cyberpunk, never startup-gradient.

### Aesthetic Direction
Dark wine-black surfaces tinted toward the crimson accent hue (OKLCH, hue ~20), with a warm-paper light theme as a first-class alternate. Vivid crimson/ruby `oklch(0.635 0.205 22)` as the only signal color; green for success lamps, orange `oklch(0.75 0.175 62)` for danger/YOLO so it stays clear of the accent. Every color is a CSS variable so both themes share one structure. Hairline rules over cards. Ledgers over grids.

Type: Bricolage Grotesque (display, 700/800), Hanken Grotesk (UI/body), Fragment Mono (data, timestamps, logs — functional mono, not vibe mono). All bundled locally in `public/fonts/`.

### Design Principles
1. Ledger, not dashboard — dense ruled rows, asymmetric columns, no card grids or hero metrics.
2. Burgundy is signal — it marks the live, the active, the actionable. Never decoration.
3. Everything is real data — no placeholder, no decorative sparkline; if it's on screen it's true of the machine.
4. Local-first is part of the brand — "nothing leaves this machine" is written on the rail because it's a feature.
5. Motion only for state — pulsing lamp while running, one staggered rise on view entry, nothing else.
